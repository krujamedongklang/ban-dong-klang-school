# คู่มือการเปิดใช้งานเว็บไซต์ออนไลน์ผ่าน GitHub และ Render (Deployment Guide)

คู่มือนี้จะแนะนำขั้นตอนการนำเว็บไซต์โรงเรียนบ้านดงกลาง (ธนาคารกรุงเทพ ๓๑) ขึ้นสู่ระบบออนไลน์ฟรี เพื่อให้ทุกคนสามารถเข้ามาเยี่ยมชมได้ โดยทำงานร่วมกับ GitHub (สำหรับจัดเก็บโค้ด) และ Render (สำหรับโฮสต์เว็บไซต์และระบบปรับขนาดอัตโนมัติ)

---

## ขั้นตอนที่ 1: การอัปโหลดโค้ดขึ้น GitHub

1. สมัครใช้งานบัญชีที่ [GitHub](https://github.com) (หากยังไม่มี)
2. สร้าง Repository ใหม่บนหน้าเว็บของ GitHub:
   - กดปุ่ม **New** (หรือเครื่องหมายบวก `+` ที่มุมขวาบน)
   - ตั้งชื่อโปรเจกต์ เช่น `ban-dong-klang-school`
   - เลือกสถานะเป็น **Public** หรือ **Private** ก็ได้ตามต้องการ
   - **ไม่ต้องติ๊กเลือก** Add a README file, Add .gitignore หรือ Choose a license
   - กดปุ่ม **Create repository**
3. เปิดเครื่องมือ Terminal หรือ Command Prompt ในโฟลเดอร์โปรเจกต์เว็บไซต์ของคุณ
4. รันคำสั่งตามลำดับดังนี้เพื่อผูกโฟลเดอร์และอัปโหลดโค้ด:

```bash
# 1. เริ่มต้นระบบควบคุมเวอร์ชันด้วย Git
git init

# 2. แอดไฟล์ทั้งหมดในโฟลเดอร์เตรียมอัปโหลด (ยกเว้นไฟล์ที่เราละเว้นไว้ใน .gitignore เช่น .env และ node_modules)
git add .

# 3. บันทึกเวอร์ชันแรก
git commit -m "initial school website commit"

# 4. เปลี่ยนชื่อ Branch หลักให้เป็น main
git branch -M main

# 5. ผูกเข้ากับ GitHub Repository ที่เพิ่งสร้างใหม่ (เปลี่ยน URL ด้านล่างเป็นของบัญชีของคุณ)
git remote add origin https://github.com/[ชื่อบัญชีของคุณ]/ban-dong-klang-school.git

# 6. ดันโค้ดขึ้นระบบ GitHub
git push -u origin main
```

## ขั้นตอนที่ 2: การเปิดใช้งาน (Deploy) บน Render

Render เป็นบริการคลาวด์ที่รองรับการทำ Static Site ฟรี โหลดได้รวดเร็วมาก และจะสร้างลิงก์เว็บไซต์ในรูปแบบความปลอดภัย HTTPS (`https://...onrender.com`) ให้อัตโนมัติ

1. สมัครหรือเข้าสู่ระบบที่ [Render](https://render.com) (สามารถกดปุ่มเชื่อมต่อเพื่อสมัครผ่านบัญชี GitHub ได้โดยตรง)
2. บนหน้า Dashboard ของ Render กดปุ่ม **New +** ที่มุมขวาบน แล้วเลือก **Static Site**
3. ค้นหาและเลือก Repository `ban-dong-klang-school` ที่เราอัปโหลดขึ้น GitHub ในขั้นตอนก่อนหน้า
4. ตั้งค่าหน้าเปิดใช้งานดังนี้:
   - **Name**: `ban-dong-klang-school` (หรือชื่ออื่นตามสะดวก ชื่อนี้จะถูกนำไปตั้งเป็นชื่อโดเมน)
   - **Branch**: `main`
   - **Build Command**: `npm install && npm run build`
   - **Publish Directory**: `dist`
5. **ขั้นตอนสำคัญ: การตั้งค่าคีย์ Supabase อย่างปลอดภัย**:
   - เลื่อนลงมาด้านล่างก่อนกดสร้าง กดที่ปุ่ม **Environment Variables**
   - เพิ่มตัวแปรสิ่งแวดล้อมดังนี้:
     - **Key**: `VITE_SUPABASE_URL` | **Value**: *ใส่ URL ของคุณจาก Supabase*
     - **Key**: `VITE_SUPABASE_ANON_KEY` | **Value**: *ใส่ Anon/Public Key ของคุณจาก Supabase*
     *(ขั้นตอนนี้ทำให้เว็บไซต์ของคุณสามารถคุยกับ Supabase ได้อย่างปลอดภัยโดยไม่ต้องเอาคีย์ไปเปิดเผยไว้ใน GitHub)*
6. กดปุ่ม **Create Static Site** ด้านล่างสุด

---

## ขั้นตอนที่ 3: ตรวจสอบและอัปเดตเว็บไซต์

- ระบบของ Render จะใช้เวลาในการสร้างและจัดเตรียมเซิร์ฟเวอร์ประมาณ 1-2 นาที เมื่อเสร็จเรียบร้อยแล้ว สถานะจะขึ้นเป็น **Live** สีเขียว
- คุณสามารถคลิกลิงก์ที่ Render จัดเตรียมให้เพื่อเข้าสู่เว็บไซต์จริงได้ทันที (เช่น `https://ban-dong-klang-school.onrender.com`)
- **การอัปเดตแบบอัตโนมัติ (CI/CD)**: เมื่อใดก็ตามที่คุณแก้ไขโค้ดบนเครื่องคอมพิวเตอร์ของคุณและสั่งรันคำสั่ง `git push` โค้ดที่อัปเดตจะถูกส่งไปที่ GitHub และ Render จะดึงโค้ดนั้นมาปรับปรุงหน้าเว็บไซต์จริงให้อัตโนมัติทันทีภายใน 1 นาที โดยที่เว็บไม่มีการปิดตัว (Zero-Downtime)!
